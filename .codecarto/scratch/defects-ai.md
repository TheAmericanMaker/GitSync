## AI feature contract sketch

**Chat (`AiChatService`):**
- *Trigger*: user types into the AI tab in `ai_features_page.dart` â†’ calls `aiChatService.sendMessage(text)`.
- *Reads*: `repoman_aiProvider`, `repoman_aiApiKey`, `repoman_aiEndpoint`, `repoman_aiChatModel`, `repoman_aiToolModel` from `repoManager`; current repo's `gitProvider`, `httpAuthCredentials`, remote URL via `uiSettingsManager` and `GitManager`. Builds a `ToolContext` with `(repoIndex, repoPath, gitProvider, accessToken, owner, repo, providerManager, authorName, authorEmail)`.
- *Persists*: per-repo conversation JSON at `<applicationSupport>/ai_chats/<repoIndex>.json` after every message; in-memory `_conversations[repoIndex]` for fast switch.
- *Side effects*: invokes any registered tool (file read/write, git ops, provider ops). Confirmation routed through `onConfirmationRequired` to UI dialogs for `warn`/`confirm`/`danger` tiers. Tool calls run via `ToolExecutor` with a 60s timeout.
- *Error behavior*: errors set `conv.error` (ValueNotifier-mirrored to UI), but partial assistant messages may persist in the conversation; double-key/rate-limit/network errors all surface as opaque strings (see A2.1, A2.3, A2.6).
- *Network*: SSE streaming via `ai_stream_client.dart:streamCompletion` to one of Anthropic/OpenAI/Google/self-hosted endpoints, 120s socket timeout.
- *Loop cap*: `_maxToolRounds = 25` rounds. Two-model split: `toolModel` runs all tool-using rounds; `chatModel` runs final synthesis when models differ.

**Wand / autocomplete (`AiCompletionService`):**
- *Trigger*: invoked by commit-message-suggestion / other UI affordances (callers in `lib/ui/...` and `lib/api/manager/git_manager.dart`).
- *Reads*: same provider config (`repoman_aiProvider`, `repoman_aiApiKey`, `repoman_aiEndpoint`) plus `repoman_aiWandModel`. No `ToolContext`; pure prompt-in / text-out.
- *Persists*: nothing.
- *Side effects*: none directly; caller chooses what to do with the suggestion.
- *Error behavior*: returns `null` on **every** failure (404, 401, 429, network, parse error, empty content). No structured error â€” this is a UX hole (A1.18, A1.17).
- *Network*: single non-streaming POST, 60s timeout.

**Agent (tool-using rounds inside chat):**
- *Trigger*: only via chat â€” there is no separate agent entrypoint. Whenever the chat's tool model emits `tool_use` blocks, the agent loop dispatches them.
- *Recursion cap*: `_maxToolRounds = 25` rounds in `_runAgenticLoop`. **No cap on parallel tool calls per round** (A1.7). No cap on total tool calls per session.
- *Tool depth*: tools do not call into other tools. One layer of dispatch.
- *Side effects per tool*:
  - Read-only: `git_status`, `git_log`, `git_diff`, `git_branch_list`, `file_read`, `file_list`, `file_search`, `repo_info`, `list_issues`, `list_pull_requests`, `get_issue_detail`, `get_pr_detail`, `list_*`, `get_*`. Confirmation: none.
  - Mutating (warn): `file_edit`, `git_commit`, `git_branch_create`, `git_branch_create_from_commit`, `git_branch_rename`, `git_create_tag`, `git_gitignore_write`, `git_info_exclude_write`, `git_stage_lines`.
  - Mutating (confirm): `file_write`, `git_push`, `git_pull`, `git_branch_delete`, `git_discard`, `git_amend_commit`, `git_revert_commit`, `git_cherry_pick`, `git_fetch`, `git_sync`, `git_undo_commit`, `git_prune_corrupted_objects`, `git_set_disable_ssl`, `git_add_remote`, `git_update_submodules`, `git_abort_merge`, `git_untrack`, `create_issue`, `add_issue_comment`, `update_issue`, `close_reopen_issue`, `create_pull_request`, `create_repo`.
  - Destructive (danger): `git_reset_to_commit`, `git_squash_commits`, `git_force_pull`, `git_force_push`, `git_download_and_overwrite`, `git_upload_and_overwrite`, `git_set_remote_url`, `git_delete_remote`, `git_checkout_commit`.
